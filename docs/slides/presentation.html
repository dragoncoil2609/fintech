Act 2 · The pattern 
def make_ai():
    """AI categorizer: bedrock (Converse) | local (rule-based, offline)."""
    if config.ai_backend == "bedrock":
        return ai.BedrockAI(region=config.aws_region, model_id=config.ai_model_id)
    if config.ai_backend == "local":
        return ai.LocalAI()                       # offline · $0 · rule-based

def make_userstore():
    """sqlite | postgres | dynamodb | documentdb | mysql — chosen from env."""
    backend = config.resolved_db_backend
    if backend == "postgres":  return userstore.PostgresUserStore(url=...)
    if backend == "sqlite":    return userstore.SQLiteUserStore(db_path=...)
    if backend == "dynamodb":  return userstore.DynamoDBUserStore(table_name=...)
    # + documentdb, mysql … same interface, swapped by ONE env var

5 DB · 2 AI · 2 PDF · 2 storage — đổi bằng 1 env flag, không sửa code, không vendor lock-in. Test local SQLite+LocalAI ($0), deploy prod đổi 1 biến.

Để engineer đọc 5 giây. "Toàn bộ bí mật: 1 file, đọc env, trả implementation. handlers.py chỉ phụ thuộc interface → test cực dễ với stub local." Đây là lý do build nhanh mà vẫn deploy được prod.
Act 2 · Hybrid AI

Không phải mọi dòng đều cần LLM.

1 · Keyword fast-path
Merchant rõ ràng (Highlands, GRAB, EVN…) → khớp tức thì. $0 · offline · ~65%
→
2 · Bedrock Converse
Chỉ dòng mơ hồ mới gọi LLM. Batch 10 dòng/lần, JSON-only, prompt cache.
→
3 · LocalAI fallback
Bedrock lỗi/throttle? → tự rớt về rule-based. Upload không bao giờ chết.

Hệ quả: ~65% dòng xử lý miễn phí, chỉ ~35% chạm LLM — và service vẫn sống kể cả khi Bedrock sập.

Đây là quyết định #1 thành hình. 3 tầng: keyword (nhanh, free) → Bedrock (chỉ ambiguous) → fallback Local (chống lỗi). Nhấn: resilience + cost. "AI tốt nhất là AI bạn không phải gọi."
Act 2 · Prompt design

Single-shot · few-shot · JSON-only. (no multi-step)

SYSTEM Vietnamese transaction categorizer → exactly one of: Food, Transport, Shopping, Bills, Entertainment, Health, … Highlands, GRAB, Shopee, EVN known. Respond ONLY JSON: [{"idx": int, "category": "<Category>", "confidence": "high|medium|low"}] FEW-SHOT 1. HIGHLANDS COFFEE BUI VIEN | -65000 → Food / high 2. SALARY JUNE 2026 | 28000000 → Salary / high 3. EVN HCMC TIEN DIEN T5 | -850000 → Bills / high

Versioned (PROMPT_VERSION) · batch 10 rows/call · system + few-shot tách riêng để Bedrock prompt-cache tái dùng (giảm token).

"JSON-only, một lần gọi, không vòng lặp." Few-shot dạy merchant VN. cachePoint tách phần system tĩnh (cache) khỏi dữ liệu động → rẻ hơn mỗi lần gọi. Prompt được version hoá để A/B.
Act 2 · Data integrity

Dedup 4 lớp — vì sao bank export hay trùng.

File hash — upload đúng file cũ → chặn ngay (DuplicateFileError).
Transaction fingerprint — amount + mô tả chuẩn hoá (không tính ngày), scope theo user.
Date tolerance — cùng giao dịch lệch 1–2 ngày giữa các lần export → nhận ra.
Manual warn — nhập tay trùng → cảnh báo mềm, người dùng tự quyết.

Mọi lớp scope theo user_id — 2 người upload cùng file vẫn độc lập. Silent skip = bad UX, nên ta cảnh báo thay vì âm thầm bỏ.

Quyết định #6. Giải thích vì sao quan trọng: bank export tháng này chồng tháng trước, dễ nhân đôi chi tiêu → sai insight. 4 lớp, đều theo user. Nhấn 'manual warn' = tôn trọng người dùng.
Act 2 · Scale

File lớn không block UI — async qua SQS.

flowchart LR
  U["File larger than 1MB"] --> API["POST /enqueue"]
  API --> S3["S3 (raw)"]
  API --> Q["SQS (QUEUED)"]
  Q --> W["Fargate worker
process_job · idempotent"]
  W --> DB[("PostgreSQL")]
  W -. "on failure" .-> DLQ["DLQ"]
  FE["Frontend"] -. "poll /job-status" .-> API
  • Sync cho file nhỏ — phản hồi tức thì.
  • Async cho file lớn — không treo request.
  • Worker idempotent → SQS gửi lại vẫn an toàn.
  • Lỗi độc → DLQ, không kẹt hàng đợi.
  • FE poll /job-status: QUEUED→PROCESSING→COMPLETED.
Quyết định #4. "UX nhỏ cần tức thì; file lớn cần an toàn." Worker idempotent là chìa khoá — at-least-once delivery của SQS không gây nhân đôi. DLQ để debug poison message.
Act 2 · Security

Multi-tenant ngay từ đầu — không phải afterthought.

Identity

Cognito User Pool · idToken JWT verify server-side qua JWKS (RS256, kiểm iss/aud/token_use). Có guest mode + cờ REQUIRE_AUTH.

Isolation

Mọi truy vấn scope theo user_id = Cognito sub. Dedup, catalog, cost — đều per-user. Hai người không thấy data của nhau.

Network

API trong private subnet, ra AWS qua VPC endpoints (PrivateLink). WAFv2 trước ALB + CloudFront. Secrets ở Secrets Manager.

Abuse control

Rate limiting per-user: ElastiCache Valkey phân tán (đa task) + in-memory local, fail-open nếu cache sập.

Cho audience business/security: "không phải prototype hở. JWT verify thật ở server, không tin client. Per-user isolation. Rate limit phân tán." Đây là điểm tin cậy cho PM/security.

Act 4 · 3 lessons
01

AI ẩn là AI tốt

User chỉ thấy "không phải nhập sổ" — không biết Bedrock chạy phía sau. Magic, không phải máy móc.

02

Adapter = tự do

5 DB + 2 AI + 2 PDF đổi bằng 1 env flag. Test local $0, deploy prod không sửa dòng nào.

03

Cost-aware ngày 0

Haiku + keyword hybrid + prompt cache = ~$0.017/1k tx. Đừng bắt đầu bằng Sonnet.

Nếu làm lại: merchant catalog cache — skip Bedrock cho merchant đã thấy ≥3 lần (confidence ≥0.8) → tiết kiệm thêm 70–80% LLM calls sau 1–2 tháng.

3 takeaway chính. Câu "nếu làm lại" cho thấy team nghĩ về roadmap, khiêm tốn. Đây là phần khán giả nhớ lâu nhất.
